# Data and AI CV Template

## Target Roles

Data scientist, machine learning engineer, AI engineer, data analyst, analytics engineer, applied scientist, LLM/NLP/computer vision specialist.

## When to Use

Use when the JD emphasizes modeling, analytics, experimentation, data pipelines, model evaluation, business impact from data, or production ML/AI systems.

## Section Order

1. Header
2. Data/AI summary
3. Technical skills
4. Professional experience
5. Selected projects/publications/portfolio
6. Education/certifications

## Reconstructed Skeleton

### Summary

Data/AI practitioner with <years> years applying <analytics/ML/AI> to <domain>. Experienced in <Python/SQL/ML stack>, <data systems>, and translating models or insights into <business/product/operational outcome>.

### Technical Skills

- Languages/query: Python, SQL, R, Scala
- ML/AI: scikit-learn, PyTorch, TensorFlow, LLMs, NLP, computer vision, recommendation systems
- Data: ETL/ELT, data quality, feature engineering, warehouses, orchestration
- Evaluation: experimentation, A/B testing, model metrics, dashboards, monitoring

### Experience Bullet Pattern

- Developed <model/analysis/pipeline> for <business/product problem>, improving <metric> by <number>.
- Built <data workflow/dashboard> used by <audience>, enabling <decision/action> and reducing <manual work/risk>.
- Evaluated <model/experiment> with <metric/method>, leading to <launch/rollback/optimization decision>.

## Bullet Style

Connect technical work to decisions, product behavior, revenue/cost, risk, or operational improvement. Include model/data scale only when relevant and explainable.

## Optimization Goal

Show applied data judgment: problem framing, data quality, methods, evaluation, deployment/adoption, and measurable impact.

## Pitfalls

- Listing algorithms without problems or outcomes.
- Presenting academic projects as production experience.
- Omitting data cleaning, evaluation, and deployment details.
- Using AI buzzwords without evidence.
