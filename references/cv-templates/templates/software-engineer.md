# Software Engineer CV Template

## Target Roles

Backend engineer, frontend engineer, full-stack engineer, platform engineer, infrastructure engineer, senior software engineer.

## When to Use

Use when the JD emphasizes shipping software, system design, code quality, performance, reliability, APIs, infrastructure, or product engineering impact.

## Section Order

1. Header
2. Targeted summary
3. Technical skills
4. Professional experience
5. Selected projects or open-source work
6. Education/certifications

## Reconstructed Skeleton

### Header

Name · Location/remote preference · Email · Phone · LinkedIn · GitHub/portfolio

### Targeted Summary

Software engineer with <years> years building <systems/products>. Strong in <primary stack>, <domain>, and <impact area>. Recent work includes <credible achievement tied to JD>.

### Technical Skills

- Languages: <languages used professionally>
- Backend/platform: <frameworks, APIs, databases, cloud, infra>
- Frontend/mobile: <only if relevant>
- Practices: testing, observability, CI/CD, performance, security, accessibility

### Experience Bullet Pattern

- Built <system/feature> for <users/team/business>, improving <metric/result> by <number> using <tech/context>.
- Reduced <latency/cost/error rate/manual work> by <number> through <specific engineering action>.
- Led <technical initiative> across <team/scope>, resulting in <release/adoption/reliability outcome>.

## Bullet Style

Use specific action verbs, production context, relevant stack terms, and measurable outcomes. For senior roles, add ownership scope, trade-offs, architecture, mentoring, or incident/reliability examples.

## Optimization Goal

Show evidence of engineering judgment, production ownership, measurable impact, and stack match with the JD.

## Pitfalls

- Listing every tool ever used.
- Writing responsibility bullets without outcomes.
- Claiming seniority without system ownership, mentoring, or trade-off examples.
- Adding JD keywords that cannot be defended.
