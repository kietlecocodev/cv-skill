# Skills Section

## Purpose

Make relevant hard skills easy for ATS parsers and recruiters to find without creating a noisy skill dump.

## What Good Looks Like

Group skills by category and order categories by JD relevance. Use exact JD terminology when truthful, then support important skills in experience bullets.

## Weak vs Strong

Weak: JavaScript, Python, leadership, teamwork, AI, cloud, databases, hard-working, communication.

Strong: Languages: TypeScript, Python, SQL. Backend: Node.js, REST APIs, PostgreSQL, Redis. Cloud/DevOps: AWS, Docker, GitHub Actions. Practices: testing, observability, performance optimization.

## Role-Specific Notes

- Engineering: group languages, frameworks, data stores, cloud, quality practices.
- Product: group product discovery, analytics, experimentation, roadmap, stakeholder tools.
- Data/AI: group languages, ML libraries, data platforms, evaluation, visualization.
- Management: group leadership practices, delivery systems, technical strategy, people systems.

## Common Mistakes

- Adding tools used only once in tutorials.
- Mixing soft traits with hard skills.
- Listing every technology from a long career.
- Failing to repeat important skills in experience context.

## Source Patterns

Indeed and Jobscan emphasize job-description keywords and standard readable sections; Resume Worded emphasizes tailored technical skills. See `../sources.md`.
