# Software Engineer Keyword Map

## Target Role Family

Backend, frontend, full-stack, platform, infrastructure, senior software engineering.

## How to Use

Use these keywords only when they truthfully match the candidate's experience or the target JD. Prefer exact JD terms when truthful, then support important keywords in experience bullets.

## Core Hard-Skill Keywords

- Programming: JavaScript, TypeScript, Python, Java, Go, C++, Rust, SQL
- Backend: REST APIs, GraphQL, microservices, distributed systems, event-driven architecture, authentication, authorization
- Frontend: React, Vue, Angular, Next.js, accessibility, design systems, state management, performance optimization
- Data/storage: PostgreSQL, MySQL, Redis, MongoDB, Elasticsearch, data modeling, caching
- Cloud/infrastructure: AWS, GCP, Azure, Docker, Kubernetes, Terraform, CI/CD
- Quality: unit testing, integration testing, observability, logging, monitoring, incident response

## Domain/System Keywords

distributed systems, API design, scalability, reliability, security, developer productivity, platform engineering

## Leadership or Collaboration Keywords

code review, mentoring, technical design, cross-functional delivery, incident communication

## Evidence Verbs

Built, shipped, optimized, migrated, refactored, automated, scaled, debugged, led, designed, maintained.

## Misuse Warnings

- Do not list skills the candidate cannot defend in interviews.
- Do not add terms only because they appear in the JD.
- Do not confuse exposure with ownership.
- Do not stuff keywords without evidence in experience or projects.
