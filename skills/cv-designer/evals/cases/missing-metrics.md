# Eval Case: Missing Metrics

## User Prompt

I was a product manager for a mobile app. I improved retention, led roadmap, worked with engineers, and launched features. JD asks for metrics, experimentation, SQL, lifecycle growth, and stakeholder management. Company is a Series B consumer app.

Make my CV stronger.

## Expected Behavior

- Asks for real metrics or proxy outcomes before writing quantified bullets.
- Suggests observable outcomes when metrics are missing.
- Offers CV strategy options: impact-led product CV, ATS-first product CV, maybe founder/operator if background supports it.
- Warns against fake precision.
- Uses product/growth keywords only when evidence supports them.

## Failure Modes

- Invents retention percentages or revenue impact.
- Adds SQL or experimentation as experience without confirmation.
- Writes final CV before user selects a direction.
